# v0 プロンプト集 (v0_PROMPT.md)

参加者をチームにランダム振り分けする「くじ引き」Webアプリの UI を [v0](https://v0.dev) で生成するためのプロンプト集。

方針: **配色値はデザインツール（v0）に委ね、役割・制約・体験を指定する**（[DESIGNS.md](./DESIGNS.md) §5.1 と整合）。

運用メモ:
- v0 は **1プロンプト1画面** が安定。下記は画面ごとに分割している。
- 参加者画面 →（同チャットで続けて）会場表示 → 管理者 の順に出すと、デザインシステムが引き継がれ統一感が保てる。
- 生成後、v0 が吐いた CSS 変数を DESIGNS.md §5.1 の暫定トークン値の置き換えに使うと、実装に落とし込みやすい。

---

## 1. 参加者画面（最初に生成・体験の核）

```text
「参加者をチームにランダム振り分けする"くじ引き"Webアプリ」の参加者画面を作って。
Next.js (App Router) + Tailwind + shadcn/ui で、まずUIのみ（モックデータ可）。

# トーン
イベント会場で映える、くじ引きのワクワク感のある洗練されたデザイン。
スマホ前提のモバイルファースト（コンテナ幅 約480px、中央寄せ）。

# 配色とトークン（重要）
- 具体的な色はあなたが提案してOK。ただし必ず CSS Custom Properties で
  「役割ベースのトークン」として定義し、色はそこを上書きするだけで差し替え可能にする。
- 役割: bg / surface / surface-muted / border / text / text-muted /
  primary / primary-strong / accent / success / danger / warning。
- 単色（青だけ）に依存しない、役割で整理されたパレットにする。
- ライト/ダーク両対応（OS設定追従 + 手動切替トグル）。ダークは純黒を避け濃紺寄り。

# 必須要素（情報の優先順位順）
1. ヒーロー的な大きな「くじを引く」ボタン（誘目・押した瞬間の触感的フィードバック）
2. 自分の結果カード（決定チーム名を大きく祝祭的に表示）
3. 全体進捗（参加人数/総枠・残り枠を「数値＋プログレスバー」で二重表示）
4. チーム一覧（各チームに色相を割り当てつつ、必ずチーム名ラベルを併記＝色だけに依存しない）
- 上部にタイトルとイベントコード、表示名の入力フィールド。

# くじ演出（体験の核）
- ボタンを押すと「抽選スロット」: 80ms程度でチーム名が高速に切り替わり1.5秒ほど回転 → 1つに確定。
- 確定時に pop（拡大）アニメ +「決定！」ラベル + 紙吹雪（confetti）。
- prefers-reduced-motion 設定時は演出を抑制する。

# アクセシビリティ / 視認性
- WCAG AA コントラスト（ライト/ダーク双方）。フォーカスリング可視。
- タップ領域は最小44px。フォントは clamp() で画面幅に追従。
- 色だけで状態を伝えない（ラベル/アイコンを併用）。
- 文言は日本語。チーム名は佐賀弁テーマ（例: ぎゅうにゅう班 など）のダミーで可。
```

## 2. 会場表示画面（参加者画面の後、同チャットで続けて）

```text
同じデザインシステム・トークンのまま、会場の大画面・遠距離視認向けの表示画面を作って。
- コンテナ幅を広げる（約1024px）。
- タイトルは脈動（pulse）アニメでライブ感を出す。
- 参加用のQRコードとURLを大きく表示（QRは暗背景でも読めるよう白パディング付き白地）。
- チーム一覧をメンバー名まで遠くから読める大きさで表示。各チームに色相を割り当てつつラベル併記。
- コントラスト最大化。会場の照明に応じてライト/ダークを選べるトグル。
- 自動更新時のちらつき防止（差分更新＋短いトランジション）を想定したUIに。
```

---

## 3. 管理者画面（同チャットで続けて）

```text
同じトークンのまま管理者画面を作って。情報密度高めをセクション見出しで整理。
- 現在のチーム名一覧。
- イベント設定フォーム: イベント名 / チーム数 / 総枠（均等配分の自動反映）/ プリセット・カスタム構成。
- チーム名再生成ボタン（割当0人時のみ活性）。
- 管理トークン入力フィールド（保持状態を可視化）。
- 危険操作「割当リセット / 全リセット」を視覚的に隔離し、二段階確認モーダル（語句タイプ確認）を付ける。
- インラインのエラー/成功メッセージ + トースト通知。
- 進捗ステータス表示。
```

## 調整したいとき

- 1枚にまとめたい場合は「上記3画面を1つのプロンプトに統合」を依頼。
- トーン（ポップ/シック等）、テーマ色の方向性、チーム名テーマ（佐賀弁以外）を変えたいときは該当プロンプトに追記する。
